# 記事執筆プロンプトテンプレート

## 目的
入力テーマに対し、読者の根源的な課題（Jobs-to-be-Done）と社会的な文脈を深く探求し、
検索意図を満たしつつ、読者の思考を覆すほどの「インサイト」を提供する。
読了後の深い納得感と具体的な行動変容を促し、著者への信頼とファン化を生み出す。

---

## 入力項目（記事作成時に指定）

| 項目 | 説明 |
|------|------|
| 記事の核となるテーマ | テーマや書き起こしたメモ・音声内容 |
| ターゲット読者 | 人物像・業務/ライフステージ・悩み・価値観 |
| 記事の最終目的（ゴール） | 読者の行動・状態変化（1つに絞る） |
| 必ず含めたいキーワード | 5つ以内 |
| 参考情報・著者の独自視点 | 一次情報・経験・データ・事例（なければ「特になし」） |
| 思考を深める「問い」の種 | 任意（例：「時間軸で捉え直すと？」「業界の常識への反論は？」） |
| 制約とNG | 避けるべき表現・内容 |
| KPI | 任意 |

---

## 執筆の基本要件

- 文字数: 3000〜5000文字
- 文体: 丁寧な「です・ます調」
- 論理的に違和感のない文章・構成

---

## 思考原則

### インサイトの探求
- 表面的な事象（What）ではなく「なぜ（Why）」「だからどうなる（So What）」を問う
- JTBD理論に基づき、読者が本当に片付けたい「用事」を定義

### クリティカル・シンキング
- 「その根拠は何か？」「反証可能性は？」「見過ごされた論点はないか？」

### アナロジー思考
- 別の業界や歴史上の出来事から教訓や新たな切り口を応用

### 第一印象の科学
- 損失回避／カリギュラ／バーナム効果を応用したタイトルとリード文

### エンゲージメント
- 短文・リズム・PREP法
- 読者への問いかけや共感を呼ぶミニストーリー（特に失敗談）

---

## 制作フロー

### Phase 1: 探求と戦略立案（内部処理）

**Step 1-1: 問いの再定義とリサーチ**
1. 根本課題の特定: ターゲット読者が本当に解決したい「根本的な課題（JTBD）」を言語化
2. 多元的リサーチ:
   - SERP分析: 上位10記事の主張、論拠、構成、CTAを分析
   - 信頼ソースの探索: 公的機関の統計、調査レポート、学術論文の要旨

**Step 1-2: 批判的分析とインサイト抽出**
1. ギャップ分析: 競合記事が見過ごしている論点、論拠の弱い部分、最新性を欠く情報
2. インサイトの創出: 複数のファクトを組み合わせ「キラーインサイト」を3つ仮説立て
3. 戦略決定: 最も強力なインサイトを1つ選び、記事の核として採用

### Phase 2: 執筆と最適化

**Step 2-1: 全体構造の設計**
- 構成フレーム: PREP法などを参考に説得力のある論理展開を選択
- プロット作成: 読者の感情曲線（共感→問題意識→発見→納得→行動）を意識した見出し構成

**Step 2-2: タイトルとリード文**
- タイトル案×3: 異なる切り口（課題解決型、常識破壊型、未来予測型）
- タイトル: 13文字以内、サブタイトル: 25文字以内
- リード文（150字以内）: 「本記事は、」で始め、何を解決できるかを明確に

**Step 2-3: 本文執筆**
- 可読性: 一文は短く、専門用語は避け、対話するように
- 説得力: 主張の直後に客観的データを「（出典：〇〇, 〇〇年）」と明記
- 独自性: キラーインサイトを比喩や具体例で解説、「限界」「注意点」も誠実に言及
- 表現の柔軟性: 複雑な情報は箇条書きやシンプルな表を活用

**Step 2-4: CTAとファン化**
- ゴールに沿った具体的なCTAを提示
- 行動への最後のハードルを下げるマイクロコピー
- 著者の視点や想い（Why）を短く語り、継続的な関係性を構築

**Step 2-5: 最終レビュー**
- 全ての主張に根拠はあるか？ 日付や数字は正確か？
- 論理は一貫しているか？
- 読者の心に響く、熱量のある文章になっているか？
